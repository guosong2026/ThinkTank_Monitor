# ThinkTank Monitor 邮件发送配置说明

## 快速开始

### 步骤1：获取SMTP授权码
获取发件邮箱的SMTP授权码（见下文详细说明）。

### 步骤2：配置环境变量
**推荐方法**：使用 .env 文件
```bash
# 复制模板文件
cp .env.example .env

# 编辑 .env 文件，填入实际配置
# 1. 设置 SENDER_EMAIL（您的发件邮箱）
# 2. 设置 SENDER_PASSWORD（您的SMTP授权码）
# 3. 设置 RECIPIENT_EMAILS（收件人邮箱，多个用逗号分隔）
```

**备选方法**：命令行设置环境变量
```bash
# Windows
set SENDER_EMAIL=your_email@outlook.com
set SENDER_PASSWORD=your_smtp_auth_code
set RECIPIENT_EMAILS=recipient1@example.com

# Linux/macOS
export SENDER_EMAIL=your_email@outlook.com
export SENDER_PASSWORD=your_smtp_auth_code
export RECIPIENT_EMAILS=recipient1@example.com
```

### 步骤3：测试邮件发送
运行测试命令验证配置：
```bash
python email_sender.py --test
```

### 步骤4：启动监控
启动Web界面或运行监控程序：
```bash
# 启动Web界面（推荐）
python app.py
# 或
start_web.bat  # Windows
./start_web.sh  # Linux/macOS

# 运行命令行监控
python monitor.py --multi
```

## 概述

ThinkTank Monitor 提供了邮件通知功能，当监控到新报告时会自动发送邮件通知。要启用此功能，您需要正确配置SMTP（邮件发送）设置。

## 配置方式（推荐使用环境变量）

由于安全考虑，ThinkTank Monitor v1.2+ 版本已移除硬编码的邮箱配置，必须通过以下方式配置：

### 方式一：使用环境变量（推荐）
通过设置环境变量来配置邮件参数，无需修改代码文件，便于在不同环境使用不同邮箱。

### 方式二：使用 .env 文件（最推荐）
创建 `.env` 文件存储配置，既安全又方便管理，支持版本控制（.env文件不应提交到代码库）。

**注意**：不再支持直接修改 `email_sender.py` 文件中的配置，因为硬编码的默认值已被移除。未设置环境变量时，系统会提示您需要配置。

## 配置项详解

以下是 ThinkTank Monitor 支持的所有邮件配置项，这些配置必须通过环境变量或 .env 文件设置。

### 1. SENDER_EMAIL - 发件人邮箱
**作用**：用于发送邮件的邮箱地址。

**环境变量名**：`SENDER_EMAIL`

**示例**：
```bash
# .env 文件或命令行
SENDER_EMAIL=your_email@outlook.com
```

**注意**：
- 必须使用支持SMTP协议的邮箱
- 建议使用常用邮箱服务商（Outlook、Gmail、QQ邮箱等）
- **必须设置**此变量，否则邮件发送功能将无法工作

### 2. SENDER_PASSWORD - SMTP授权码
**作用**：用于SMTP服务器认证的密码或授权码。

**环境变量名**：`SENDER_PASSWORD`

**示例**：
```bash
# .env 文件或命令行
SENDER_PASSWORD=AbC123dEf456gHi7
```

**重要**：
- 不要使用邮箱登录密码
- 必须使用SMTP授权码（获取方法见下文）
- **必须设置**此变量，否则邮件发送功能将无法工作
- 请妥善保管授权码，不要泄露

### 3. SMTP_SERVER - SMTP服务器地址
**作用**：邮件服务提供商的SMTP服务器地址。

**环境变量名**：`SMTP_SERVER`

**默认值**：`smtp.office365.com`（如果未设置）

**示例**：
```bash
# .env 文件或命令行
SMTP_SERVER=smtp.office365.com
```

### 4. SMTP_PORT - SMTP端口
**作用**：SMTP服务器的端口号。

**环境变量名**：`SMTP_PORT`

**默认值**：`587`（如果未设置）

**示例**：
```bash
# .env 文件或命令行
SMTP_PORT=587
```

**常用端口**：
- 587：TLS加密端口（推荐）
- 465：SSL加密端口
- 25：普通SMTP端口（可能被屏蔽）

### 5. RECIPIENT_EMAILS - 收件人邮箱列表
**作用**：接收邮件通知的邮箱地址列表。

**环境变量名**：`RECIPIENT_EMAILS`

**格式**：多个邮箱用逗号分隔

**示例**：
```bash
# .env 文件或命令行
RECIPIENT_EMAILS=recipient1@example.com,recipient2@example.com,recipient3@example.com
```

**注意**：
- 多个邮箱用逗号分隔，不要有空格
- 支持任意数量的收件人
- **必须设置**此变量，否则邮件发送功能将无法工作

## 使用 .env 文件配置（最推荐）

使用 `.env` 文件是最安全、最方便的配置方式，它将所有敏感信息存储在一个文件中，便于管理且不会泄露到代码库中。

### 步骤1：创建 .env 文件

在 ThinkTank Monitor 项目根目录（与 `app.py` 同一目录）中创建名为 `.env` 的文件。

**快速创建方法**：
```bash
# 复制模板文件（推荐）
cp .env.example .env

# 或者手动创建空文件
touch .env  # Linux/macOS
type nul > .env  # Windows
```

### 步骤2：编辑 .env 文件

用文本编辑器打开 `.env` 文件，填入您的实际配置：

```bash
# SMTP服务器配置
SMTP_SERVER=smtp.office365.com
SMTP_PORT=587

# 发件人配置（必须设置）
SENDER_EMAIL=your_real_email@outlook.com
SENDER_PASSWORD=your_real_smtp_authorization_code

# 收件人配置（必须设置）
RECIPIENT_EMAILS=recipient1@example.com,recipient2@example.com

# 可选配置
# EMAIL_SUBJECT_PREFIX=【新报告监控】
# CHECK_INTERVAL_HOURS=1
# MONITOR_ENABLED=1
```

### 步骤3：确保 .env 文件被忽略

为了安全，请确保 `.env` 文件不被提交到代码库：

1. 检查项目根目录是否有 `.gitignore` 文件
2. 确保 `.gitignore` 文件中包含以下行：
   ```
   .env
   .env.local
   .env.*.local
   ```

### 步骤4：启动应用

配置完成后，正常启动应用即可：

```bash
# 启动Web界面
python app.py

# 或者使用启动脚本
start_web.bat  # Windows
./start_web.sh  # Linux/macOS
```

### .env 文件的好处

1. **安全性高**：敏感信息不在代码中，不会意外提交到版本控制系统
2. **管理方便**：所有配置集中在一个文件中，便于修改和维护
3. **环境隔离**：可为不同环境（开发、测试、生产）创建不同的 .env 文件
4. **无需记忆命令**：无需每次启动都设置环境变量

### 常见问题

**Q：.env 文件应该放在哪里？**
A：放在项目根目录，与 `app.py` 同一目录。

**Q：如何为不同环境创建不同的配置？**
A：可以创建多个 .env 文件，如 `.env.development`、`.env.production`，启动时指定加载哪个文件。

**Q：.env 文件中的变量名可以自定义吗？**
A：不可以，必须使用预定义的变量名：`SMTP_SERVER`、`SMTP_PORT`、`SENDER_EMAIL`、`SENDER_PASSWORD`、`RECIPIENT_EMAILS`。

**Q：修改 .env 文件后需要重启应用吗？**
A：是的，修改配置后需要重启应用才能生效。

## 环境变量配置方式（命令行）

如果您不想使用 .env 文件，也可以通过命令行直接设置环境变量：

### Windows (PowerShell)
```powershell
# 设置环境变量
$env:SMTP_SERVER="smtp.office365.com"
$env:SMTP_PORT="587"
$env:SENDER_EMAIL="your_email@outlook.com"
$env:SENDER_PASSWORD="your_smtp_password"
$env:RECIPIENT_EMAILS="email1@example.com,email2@example.com"

# 启动应用
python app.py
```

### Windows (CMD)
```cmd
set SMTP_SERVER=smtp.office365.com
set SMTP_PORT=587
set SENDER_EMAIL=your_email@outlook.com
set SENDER_PASSWORD=your_smtp_password
set RECIPIENT_EMAILS=email1@example.com,email2@example.com
python app.py
```

### Linux/macOS (Bash)
```bash
export SMTP_SERVER=smtp.office365.com
export SMTP_PORT=587
export SENDER_EMAIL=your_email@outlook.com
export SENDER_PASSWORD=your_smtp_password
export RECIPIENT_EMAILS="email1@example.com,email2@example.com"
python app.py
```

**优先级**：环境变量 > 配置文件修改

## 如何获取SMTP授权码

### 为什么要使用SMTP授权码？
SMTP授权码（也称为应用密码）是专门为第三方应用生成的密码，相比使用邮箱登录密码更安全：
1. **安全性更高**：即使授权码泄露，攻击者也无法登录您的邮箱
2. **权限可控**：可以随时撤销特定应用的授权码
3. **便于管理**：可为不同应用生成不同的授权码

### Outlook邮箱获取方法
1. 登录 [Outlook官网](https://outlook.live.com)
2. 点击右上角头像 → **查看账户**
3. 选择 **安全** → **高级安全选项**
4. 在"应用密码"部分，点击 **创建新应用密码**
5. 输入应用名称（如"ThinkTankMonitor"）
6. 复制生成的16位授权码
7. **立即保存到 .env 文件或环境变量中**（授权码只显示一次！）

### QQ邮箱获取方法
1. 登录QQ邮箱网页版
2. 点击 **设置** → **账户**
3. 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"
4. 开启 **IMAP/SMTP服务**
5. 根据提示发送短信验证
6. 获取16位SMTP授权码

### 163邮箱获取方法
1. 登录163邮箱网页版
2. 点击 **设置** → **POP3/SMTP/IMAP**
3. 开启 **IMAP/SMTP服务**
4. 根据提示获取授权码

### Gmail邮箱获取方法
1. 登录Gmail账户
2. 访问 [应用密码页面](https://myaccount.google.com/apppasswords)
3. 选择 **应用**（如"邮件"）和设备
4. 点击 **生成**
5. 复制生成的16位密码

### 配置说明
**重要**：ThinkTank Monitor v1.2+ 版本不再支持在代码中硬编码配置。获取授权码后，必须通过以下方式配置：

1. **推荐方法**：将授权码添加到 `.env` 文件中：
   ```bash
   # 在 .env 文件中
   SENDER_PASSWORD=AbC123dEf456gHi7  # 替换为您的授权码
   ```

2. **备选方法**：通过命令行设置环境变量：
   ```bash
   # Windows
   set SENDER_PASSWORD=AbC123dEf456gHi7
   
   # Linux/macOS
   export SENDER_PASSWORD=AbC123dEf456gHi7
   ```

### 授权码使用注意事项
1. **立即保存**：授权码生成后立即保存到 `.env` 文件或环境变量中
2. **妥善保管**：不要将授权码分享给他人，不要提交到代码库
3. **定期更换**：建议每3-6个月更换一次授权码
4. **及时撤销**：不再使用的应用及时撤销其授权码
5. **文件保护**：确保 `.env` 文件被 `.gitignore` 忽略，防止意外提交

## 测试配置

完成环境变量配置后，建议先测试邮件发送功能：

### 测试步骤
1. **确保配置已设置**：检查 `.env` 文件或环境变量已正确配置
2. **运行测试**：
   ```bash
   python email_sender.py --test
   ```
3. **检查结果**：
   - 如果看到"测试邮件发送成功！"，说明配置正确
   - 如果看到错误信息，请根据提示检查配置

### 配置检查
在运行测试前，可以检查当前配置：

```bash
# 查看当前环境变量配置
python -c "import os; print('SMTP_SERVER:', os.environ.get('SMTP_SERVER', '未设置')); print('SENDER_EMAIL:', os.environ.get('SENDER_EMAIL', '未设置')); print('RECIPIENT_EMAILS:', os.environ.get('RECIPIENT_EMAILS', '未设置'))"
```

### 常见错误及解决方法

#### 错误1：未设置必需的环境变量
**错误信息**："发件人邮箱未设置" 或 "收件人邮箱列表为空"
**解决方法**：
1. 检查是否设置了所有必需的环境变量：`SENDER_EMAIL`、`SENDER_PASSWORD`、`RECIPIENT_EMAILS`
2. 确认 `.env` 文件位于正确位置（项目根目录）
3. 重启终端或应用使环境变量生效

#### 错误2：535 Authentication Failed
**原因**：用户名或密码错误
**解决方法**：
1. 检查 `SENDER_EMAIL` 和 `SENDER_PASSWORD` 是否正确
2. 确认使用的是SMTP授权码，而不是邮箱登录密码
3. 确保邮箱已开启SMTP服务

#### 错误3：Connection refused
**原因**：SMTP服务器或端口错误
**解决方法**：
1. 检查 `SMTP_SERVER` 和 `SMTP_PORT` 配置
2. 尝试其他端口（如587改为465）
3. 检查网络连接和防火墙设置

#### 错误4：Timeout
**原因**：网络问题或服务器不可达
**解决方法**：
1. 检查网络连接
2. 尝试更换网络环境
3. 检查防火墙是否阻止SMTP连接

### 测试成功后的操作
1. **检查收件箱**：确认测试邮件已收到
2. **检查垃圾邮件**：如果没收到，检查垃圾邮件箱
3. **正式运行**：测试成功后，可以运行监控程序
4. **Web界面验证**：启动Web界面，在设置页面确认邮箱配置已生效

## 故障排除

### 问题1：SMTP连接失败
**错误信息**：`SMTPAuthenticationError` 或 `Connection refused`

**解决方法**：
1. 检查SMTP服务器地址和端口是否正确
2. 确认邮箱和授权码无误
3. 确保网络可以访问SMTP服务器
4. 尝试使用其他端口（587 → 465）

### 问题2：邮件发送成功但未收到
**可能原因**：
1. 邮件被放入垃圾箱/垃圾邮件文件夹
2. 收件人邮箱地址错误
3. 邮件被邮件服务商拦截

**解决方法**：
1. 检查垃圾邮件文件夹
2. 验证收件人邮箱地址
3. 使用其他邮箱作为收件人测试

### 问题3：配置修改后不生效
**可能原因**：
1. 未重启Web服务
2. 环境变量与配置文件冲突
3. 配置文件保存错误

**解决方法**：
1. 重启Web服务：停止后重新运行 `python app.py`
2. 检查环境变量设置
3. 确认配置文件已保存

### 问题4：授权码无效
**可能原因**：
1. 授权码已过期
2. 使用了邮箱登录密码而非SMTP授权码
3. 授权码被重置

**解决方法**：
1. 重新生成SMTP授权码
2. 确认使用的是SMTP授权码而非登录密码
3. 按照上述步骤获取新的授权码
